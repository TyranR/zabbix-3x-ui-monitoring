## Traffic graph model

Traffic graphs use two different classes of metrics.

### Rates

Network rate items use `Bps`:

```text
3X-UI: Network upload rate
3X-UI: Network download rate
```

These are suitable for real-time network throughput.

### Counters

Client, inbound and outbound traffic items use `B`:

```text
upload traffic
download traffic
total traffic
```

These are cumulative counters reported by 3X-UI API endpoints.

Do not mix `Bps` rate items and `B` cumulative counters on the same graph unless you explicitly want different semantics.

## Inbound total traffic

Some 3X-UI API responses do not expose a ready-made inbound `total` field. In that case the template calculates:

```text
inbound total traffic = inbound upload traffic + inbound download traffic
```

The same model is used for client total traffic.

## Database health graph

`3X-UI: Database health` uses two scales:

```text
left axis  = database file size and size change
right axis = database modification age in whole minutes
```

The template keeps the original seconds-based item for trigger logic:

```text
3xui.db.file.age
```

and uses a graph/dashboard-friendly minute-based item:

```text
3xui.db.file.age.minutes
```

This avoids `ms` noise in graph legends.

---

## Application runtime graph

`3X-UI: Application runtime` uses two scales:

```text
left axis  = application memory
right axis = application threads
```

The graph is intended for detecting memory growth, thread growth and runtime behaviour after updates.

## Web UI graph

`3X-UI: Web UI Response Time and Speed` uses Zabbix web scenario items:

```text
web.test.in[3X-UI Panel Web UI Access,,bps]
web.test.time[3X-UI Panel Web UI Access,Check 3X-UI Web UI,resp]
```

The speed item is scenario-level, while the response-time item is step-level.

If this graph has no data, make sure the Web scenario is enabled and has collected at least one cycle.

## Important LLD limitation

Zabbix classic template graphs cannot dynamically create one reusable graph with all discovered inbounds, clients, nodes or outbounds as separate lines.

LLD graph prototypes work like this:

```text
one discovered inbound = one generated graph
one discovered client  = one generated graph
one discovered node    = one generated graph
```

Aggregate graphs show sums, not individual discovered objects as separate lines.

If you need all discovered entities as separate lines on one canvas, use:

```text
- a manually created host graph after discovery;
- a dashboard widget built manually for a specific host;
- Grafana;
- another visualization layer.
```
